Completed so far:

Backend changes:
- Added `backend/models/Preset.js` to replace the old phase template concept with a preset model that stores ordered project phases, active preset status, and audit history.
- Added `backend/models/UserType.js` to normalize roles into a separate metadata collection and added `roleType` references on `User`.
- Added `backend/utils/schemaMigration.js` to automatically create default user types and migrate existing users and presets on startup.
- Updated `backend/config/db.js` to run schema migration after database connection.
- Updated `backend/controllers/authController.js` and `backend/controllers/userController.js` to use normalized role values and attach matching `UserType` records during user creation.
- Updated `backend/controllers/projectController.js` to use the active preset instead of legacy phase templates, to save preset snapshots on project approval, and to enforce verified-student checks during project creation and team invites.
- Added `backend/controllers/presetController.js` for preset CRUD, active preset selection, and active preset retrieval.
- Added `backend/routes/presets.js` and mounted `/api/presets` in `backend/server.js`.
- Added new project admin endpoints `pauseProject` and `resetProject` in `backend/controllers/projectController.js` and routes in `backend/routes/projects.js`.
- Added student verification endpoints in `backend/controllers/userController.js`, including OTP request/confirm and ID card upload / admin review paths.
- Added `backend/middleware/async.js` helper and `backend/middleware/upload.js` support for ID card file uploads.
- Removed the legacy `backend/models/PhaseTemplate.js` model.
